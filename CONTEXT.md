# ha-rezeptplaner

A Home Assistant add-on that helps a household plan their weekly meals, generate shopping lists, and cook recipes step by step — assisted by a configurable AI model.

## Language

### Planning

**WeekPlan**: A set of Meals covering one calendar week, anchored to a Monday start date. Only one WeekPlan is "current" at a time; older plans are archived.
_Avoid_: meal plan, schedule, menu

**Meal**: A single slot within a WeekPlan — identified by a day and a meal type (lunch or dinner) — that has been assigned a Recipe. A Meal may be a Leftovers Meal. A normal Meal carries a `portion_multiplier` (default 1) that scales how many portions of its Recipe must be cooked.
_Avoid_: dish, entry, slot

**Leftovers Meal**: A Meal that reuses the Recipe from a directly preceding normal Meal in the same WeekPlan. Marked explicitly; not generated by the AI. Can be created manually before the WeekPlan is confirmed by "doubling" a slot. May reference the source Meal via `source_meal_id` in addition to `source_recipe_name`.
_Avoid_: remainder, carryover, repeat

**SkippedSlot**: A slot within a WeekPlan that has been deliberately left empty. It has no Recipe, produces no ShoppingList entries, and is shown in the unconfirmed WeekPlan so it can later be filled via a Swap.
_Avoid_: skipped meal, empty slot, omitted slot

**PortionMultiplier**: A factor on a normal Meal that scales the Recipe's ingredient amounts for the ShoppingList. Default is 1; "doubling" a following slot raises the preceding normal Meal's multiplier to 2.
_Avoid_: scaling factor, amount factor, portion count

### Recipes

**Recipe**: The full cooking specification for a dish — name, ingredients, steps, serving count, and nutrition per serving.
_Avoid_: dish, food item

**Ingredient**: One item in a Recipe's ingredient list, with a German name, an optional English name (used for nutrition lookup), amount, unit, and category. Categories are a fixed vocabulary: Gemüse & Obst, Fleisch & Fisch, Milchprodukte & Eier, Getreide & Backwaren, Konserven & Trockenwaren, Gewürze & Öle, Sonstiges.
_Avoid_: item, food, component

**UserRecipe**: A Recipe saved manually by the user, independent of AI generation. Can be placed into a WeekPlan via a swap.
_Avoid_: saved recipe, custom recipe, favourite

### Feedback & Preferences

**Settings**: The household's cooking preferences — number of persons, diet types, disliked foods, favourite foods, maximum cooking time, budget level, and spicy preference. Persisted and fed to the AI on every generation.
_Avoid_: preferences, configuration, profile

**Rating**: A user's numeric score (1–10) for a named Recipe. Low-rated recipes are excluded from AI suggestions; high-rated ones are preferred.
_Avoid_: review, score, feedback

**Swap**: A record that a Meal's Recipe was rejected and replaced. Stores the old recipe name and the reason (e.g. `mag_ich_nicht`, `zu_teuer`). Recent Swaps are fed to the AI to avoid repetition.
_Avoid_: replacement, substitution, rejection

### Shopping

**ShoppingList**: Ingredients aggregated across all Meals in a WeekPlan, deduplicated by name and unit, grouped by category, and sorted for easy supermarket navigation.
_Avoid_: grocery list, ingredient list

### Persistence

**PlanStore**: The single persistence seam for all app data — plans, meals, swaps, ratings, settings, and user recipes. An injectable class with an async factory (`PlanStore.init(db_path)`). The same implementation runs against a real SQLite file in production and against SQLite `:memory:` in tests.
_Avoid_: repository, DAO, database layer, database module

### Theming

**Home Assistant Theme**: A YAML-defined set of CSS custom properties that Home Assistant injects into its own UI. The Rezeptplaner does not own it, but may read selected variables when running inside HA.
_Avoid_: app theme, UI skin

**Galaxy Theme**: The specific dark, space-inspired Home Assistant Theme used by this household, defined in the HA theme YAML. It is the default visual target for the Rezeptplaner add-on.
_Avoid_: dark mode, color scheme

**Theme Variable**: A single CSS custom property exposed by Home Assistant (e.g. `--primary-color`, `--ha-card-background`). The Rezeptplaner maps a subset of these to its own design tokens, with Galaxy fallbacks for standalone use.
_Avoid_: CSS variable (too generic), token

**Accent Color**: The secondary highlight color from the Galaxy Theme (`#9b7abf`), used in the Rezeptplaner for focused controls, active tabs, and other emphasis states alongside the primary teal.
_Avoid_: highlight color, secondary color

**Success Color**: In the Galaxy Theme, the same teal as the primary color (`#0b9e9e`). The Rezeptplaner maps its success states directly to this Home Assistant theme variable.
_Avoid_: ok color, green

**Surface Stack**: The three levels of background elevation used in the Rezeptplaner UI (`--surface`, `--surface-2`, `--surface-3`). They are derived from the Galaxy Theme's single card background color rather than defined separately in the theme.
_Avoid_: layer, elevation

**Faint Text**: Very low-emphasis text used for placeholders, empty states, and inactive labels. Derived from `secondary-text-color` with reduced opacity, not from the theme's darker `disabled-text-color`, so it stays readable on dark surfaces.
_Avoid_: muted text, hint text

**Divider**: A subtle separating line in the Rezeptplaner UI. Mapped to the Home Assistant theme variable `divider-color`, with a dimmer derived variant for secondary borders.
_Avoid_: separator, line

**Theme Adapter**: A frontend module (`theme.js`) that reads the active Home Assistant theme variables from `window.parent` at startup and writes them as CSS custom properties on the Rezeptplaner iFrame root, with Galaxy fallbacks for standalone use. Theme changes at runtime require reloading the add-on.
_Avoid_: theme service, theme provider
